# Security Policy

## Supported Versions

Security fixes are applied to the latest code on the `main` branch.

If you are using an older copy of this profile, update to the latest `main` revision and verify the issue still reproduces before reporting.

## Reporting a Vulnerability

Please do **not** open public issues for security vulnerabilities.

Use GitHub's private vulnerability reporting for this repository:

- Go to the repository's [**Security**](https://github.com/jonlabelle/pwsh-profile/security) tab.
- Select **Report a vulnerability**.
- Include affected function(s), environment details, reproduction steps, and potential impact.

If private reporting is unavailable, open a minimal public issue without exploit details and request a secure contact path.

## Disclosure Process

- You will receive an acknowledgement as soon as practical.
- We will validate and triage the report, then work on a fix based on severity and impact.
- When possible, we will coordinate disclosure timing with the reporter.

Thank you for helping improve the security of this project.
